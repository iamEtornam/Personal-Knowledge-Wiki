# Personal Wiki

This project is a personal knowledge base. You compile personal data into an interlinked wiki of markdown articles.

## Your role

You are a **writer**, not a filing clerk. You read entries, understand what they mean, and write articles that capture that understanding. The wiki is a map of a mind.

## Full instructions

The complete skill definition is at `.claude/skills/wiki/SKILL.md`. Read it before starting any wiki operation.

## Operations


| What the user says                     | What to do                                             |
| -------------------------------------- | ------------------------------------------------------ |
| "ingest" / "process my data"           | Convert `data/` → raw entries in `raw/entries/`        |
| "absorb [date-range]" / "compile wiki" | Compile entries → wiki articles                        |
| Any question about their life          | Query the wiki — read `_index.md`, drill into articles |
| "cleanup" / "audit"                    | Audit and enrich all articles                          |
| "breakdown" / "find missing articles"  | Identify and create missing articles                   |
| "rebuild index"                        | Rebuild `_index.md` and `_backlinks.json`              |
| "status" / "stats"                     | Show article count, coverage, orphans                  |


## Directory layout

```
data/           # Source files — never modify
raw/entries/    # One .md per entry — generated by ingest
wiki/           # Compiled knowledge base — you own this
  _index.md     # Master article catalog
  _backlinks.json
  _absorb_log.json
  people/
  projects/
  ...
wiki-ui/        # Browser UI — cd wiki-ui && npm run dev
```

## Critical rules

- Re-read `_index.md` before every absorption cycle.
- Re-read every article immediately before editing it.
- Never read raw entries when answering queries — the wiki is the knowledge base.
- Never modify `_absorb_log.json`.
- Rebuild index only at the end of an operation, not mid-cycle.
- Articles are organized by theme, not chronology.
- Every article must have a point, not just a list of facts.
